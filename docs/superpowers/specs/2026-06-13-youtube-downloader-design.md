# YouTube 视频下载器 — 设计文档

**日期：** 2026-06-13  
**状态：** 已确认  
**语言：** Python 3.13+

---

## 概述

一个基于 tkinter 的桌面 GUI 应用，用于下载 YouTube 视频。用户输入 YouTube URL，搜索后选择分辨率，下载为 mp4/webm/mkv 格式到本地。

---

## 数据模型（models.py）

### FormatOption

单个可下载的格式（视频+音频合一，或视频流+音频流组合）。

| 字段 | 类型 | 说明 |
|------|------|------|
| `format_id` | `str` | yt-dlp 格式 ID，如 `"22"` 或 `"137+140"` |
| `resolution` | `str` | 如 `"1920x1080"` |
| `fps` | `int` | 帧率 |
| `codec` | `str` | 视频编码，如 `"avc1"` / `"vp9"` |
| `audio_codec` | `str` | 音频编码，如 `"mp4a"` / `"opus"` |
| `file_size_mb` | `float` | 文件大小（MB），未知为 `0.0` |
| `ext` | `str` | 文件扩展名（raw 扩展名，不一定是最终输出格式） |
| `note` | `str` | 备注，如 `"1080p"` |

### VideoInfo

视频完整元信息。

| 字段 | 类型 | 说明 |
|------|------|------|
| `title` | `str` | 视频标题 |
| `url` | `str` | 视频原始 URL |
| `thumbnail_url` | `str` | 缩略图 URL |
| `duration_seconds` | `int` | 时长（秒） |
| `duration_str` | `str` | 格式化时长，如 `"14:25"` |
| `uploader` | `str` | 上传者名称 |
| `view_count` | `int` | 播放次数 |
| `like_count` | `int` | 点赞数 |
| `description` | `str` | 视频简介，最多截取 200 字 |
| `formats` | `list[FormatOption]` | 所有带音频的可用格式 |

### 格式筛选规则

`downloader.py` 仅返回同时包含视频流和音频流的格式：
- 单一格式（如 `22` — 720p mp4，已包含视频+音频）
- 组合格式（如 `137+140` — 1080p 视频流 + m4a 音频流）

过滤掉 `acodec == 'none'` 或 `vcodec == 'none'` 的纯视频/纯音频流。

## 下载引擎（downloader.py）

### 核心类：`Downloader`

```
class Downloader:
    def fetch_info(self, url: str) -> VideoInfo:
        """使用 yt-dlp 提取视频信息，不下载。返回 VideoInfo。"""

    def download(self, url, format_id, output_ext, save_dir, progress_callback) -> None:
        """执行下载。通过 progress_callback 实时上报进度。"""
```

### fetch_info 实现要点

- 使用 `yt_dlp.YoutubeDL` 的 `extract_info(url, download=False)`
- 解析返回的 dict，构建 `VideoInfo` 和 `FormatOption` 列表
- 筛选带音频的格式
- 按分辨率从高到低排序
- 去重：同一分辨率保留文件大小最大的（通常是质量最好的）
- 错误处理：URL 无效、网络错误、视频不可用等均抛出明确异常

### download 实现要点

- 使用 `yt_dlp.YoutubeDL` 配置 `progress_hooks` 回调
- `outtmpl` 设置为 `{save_dir}/%(title)s.%(ext)s`
- `merge_output_format` 根据用户选择的输出格式设置（mp4/webm/mkv）
- 进度回调传入的数据结构：

```python
{
    "status": "downloading",        # "downloading" | "finished" | "error"
    "percent": 48.5,                # 百分比
    "speed_mbps": 5.2,              # 速度（MB/s）
    "downloaded_mb": 78.3,          # 已下载（MB）
    "total_mb": 162.0,              # 总大小（MB），可能为 0
    "eta_seconds": 16,              # 预计剩余秒数
    "elapsed_seconds": 15,          # 已用秒数
    "filename": "D:/Downloads/video.mp4",  # 最终文件路径
}
```

### 文件大小注意

yt-dlp 经常无法提前获取精确文件大小（尤其是自适应格式）。此时 `total_mb` 为 0，GUI 的进度条改为**不确定模式**（indeterminate），仅显示已下载大小和速度。

### 组件详细说明

#### ① URL 输入区

- **标签：** 浅灰色提示文字"请输入 YouTube 视频链接"，16pt 加粗
- **输入框：** `ttk.Entry`，宽度约 60 字符，高 40px（通过 padding 实现），字体 12pt
- **搜索按钮：** `ttk.Button`，文本 "🔍 搜索"，宽度 10 字符，与输入框等高
- **按下 Enter 键**也可触发搜索

#### ② 视频信息区

- 使用 `ttk.LabelFrame` 包裹，标题"视频信息"
- 信息以网格布局展示：
  - 左列：标题、上传者、时长
  - 右列：播放量、点赞数
  - 底行：简介（多行，`wraplength` 自适应窗口宽度，最多 3 行）

#### ③ 分辨率选择区

- 使用 `ttk.LabelFrame` 包裹，标题"选择分辨率"
- 内部用 `tk.Listbox`（或自定义 Frame 列表）展示每行：
  - 格式：`○ {分辨率}  {编码}+{音频编码}  ~{大小}MB  ({扩展名})`
  - 单选模式，点击整行选中
  - 按分辨率从高到低排序
  - 同一分辨率去重（保留文件大小最大者）

#### ④ 格式下拉框

- `ttk.Combobox`，选项：`mp4` / `webm` / `mkv`，默认 `mp4`
- 只读模式（`state='readonly'`）

#### ⑤ 下载按钮

- `ttk.Button`，文本 "⬇ 开始下载"
- 宽 30 字符，高 40px，醒目样式
- 三种状态：
  - 默认："⬇ 开始下载"（蓝色/默认主题色）
  - 搜索中："⏳ 搜索中..."（灰色，禁用）
  - 下载中："⏳ 下载中..."（灰色，禁用）
  - 完成："✅ 下载完成"（绿色，2 秒后恢复默认）

#### ⑥ 进度展示区

- 使用 `ttk.LabelFrame` 包裹，标题"下载进度"
- 下载开始前隐藏，下载中展开显示
- 包含：
  - **进度条：** `ttk.Progressbar`（determinate 或 indeterminate）
  - **百分比：** 右侧对齐
  - **速度：** `{speed:.1f} MB/s`
  - **大小：** `{downloaded:.1f} / {total:.1f} MB`
  - **时间：** 耗时 `{elapsed}` | 剩余 `{eta}`
  - **文件路径：** 最终保存路径
  - **状态文字：** "正在下载..." / "处理中..." / "下载完成 ✓"

### 线程模型

```
GUI 线程（主线程）
    │
    ├── fetch_info()  →  工作线程  →  queue  →  GUI 更新
    │    (搜索)           (后台)        (结果)
    │
    └── download()    →  工作线程  →  queue  →  GUI 进度刷新
         (下载)           (后台)        (进度)
```

- **yt-dlp 耗时操作全部在后台线程执行**，避免阻塞 GUI
- 使用 `queue.Queue` 或 `root.after()` 将结果/进度传回主线程更新 UI
- 进度轮询频率：每 100ms 更新一次 UI（足够流畅且不消耗过多资源）

### 错误处理

| 场景 | 处理 |
|------|------|
| 空 URL | 弹出提示 "请输入 YouTube 链接" |
| URL 无效 | 弹出提示 "无法解析该链接，请检查后重试" |
| 网络错误 | 弹出提示 "网络连接失败，请检查网络后重试" |
| 视频不可用 | 弹出提示 "该视频不可用（可能已被删除或设为私密）" |
| 无可下载格式 | 弹出提示 "未找到可下载的格式" |
| 下载中断 | 进度区显示 "下载失败：[错误信息]" |
| 用户取消 | 停止下载线程，清理临时文件 |

所有错误提示使用 `messagebox.showerror()` 或 `messagebox.showwarning()`。
